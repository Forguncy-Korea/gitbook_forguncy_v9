# 변경된 데이터만 전송

매개변수 추가 시 파라미 유형이 배열 유형이고 테이블 데이터인 경우 배열 항목 편집 시 추가, 수정, 삭제 등 변경된 데이터만 전송할지 여부를 선택할 수 있습니다. 체크 후 필요에 따라 추가, 수정, 삭제된 데이터만 전송하도록 설정할 수 있습니다.

이 기능은 마스터-종속 테이블에 주로 사용되며 테이블이 데이터 바인딩을 켜는 경우에만 변경 데이터만 전송하는 기능을 사용할 수 있습니다.

## 변경된 데이터만 전송

배열 파라미터의 배열 항목을 편집할 때 변경 데이터만 보내기를 선택한 후 기본 키 열을 설정할 수 있습니다. 여기에 입력된 기본 키는 명령을 디자인할 때 파라미로 사용됩니다. 편집하거나 삭제한 항목과 데이터 테이블에서 업데이트할 데이터 행을 결정하는 데 사용합니다.

여기에 입력된 기본 키가 업데이트된 데이터 테이블의 기본 키와 일치하고 대/소문자를 구분해야 합니다.&#x20;

![](<../../../../.gitbook/assets/image (1512).png>)

예를 들어 "주문 세부 정보 업데이트"라는 서버단 명령을 만든 다음 배열 형식에 대한 파라미터 "주문 내역 변경"과 기본 형식의 파라미터 "주문 ID"를 설정합니다.

![](<../../../../.gitbook/assets/image (1405).png>)

배열 형식의 파라미터를 클릭한 후 팝업 배열 항목 편집 대화 상자에서 배열 항목 추가를 클릭하고 배열 항목의 이름을 편집한 다음 기본 키 열이 ID 열로 나열된 변경 데이터만 보내기를 선택합니다.![](https://help.grapecity.com.cn/download/thumbnails/72357282/image2020-3-11_14-45-29.png?version=1\&modificationDate=1648092604000\&api=v2)

![](<../../../../.gitbook/assets/image (607).png>)



그런 다음 서버단 명령의 명령을 편집합니다. \[명령 편집] 하이퍼링크를 클릭하고 서버 명령 편집 대화 상자를 표시하고 루프 명령을 선택합니다.

루프 배열 지정을 클릭하면 팝업 대화 상자의 파라미터 목록에서 주문 ID 외에 주문 세부 정보 변경이라는 세 가지 파라미터가 있음을 확인할 수 있습니다.&#x20;

* AddRows: 이 매개 변수를 삽입하면 추가된 데이터만 전송됩니다.
* EditRows: 이 매개 변수를 삽입하면 수정된 데이터만 전송됩니다.
* DeleteRows: 이 매개 변수를 삽입하면 삭제된 데이터만 전송됩니다.

![](<../../../../.gitbook/assets/image (1793).png>)

서버단 명령이 생성되면 페이지에서 셀 범위를 선택하고 버튼을 설정합니다. 버튼의 명령을 편집하고, 명령을 선택하여 서버단 명령을 호출하고, 명령은 루프 명령을 선택합니다.

파라미터 목록에서 파라미터 "주문 세부 정보 변경" 및 "주문 ID" 값을 설정합니다.

* 값 뒤에 있는 버튼을 클릭하여 테이블에서 데이터 선택 대화 상자를 표시합니다. 테이블 이름을 선택하고 데이터 그룹에 대한 열 이름을 설정합니다. 페이지에서 테이블의 열 이름을 미리 설정해야 합니다.![](https://help.grapecity.com.cn/download/thumbnails/72357282/image2020-3-11_15-5-23.png?version=1\&modificationDate=1648092604000\&api=v2)
* 파라미터 "주문 ID"의 값은 페이지에 바인딩된 "주문 ID" 필드가 있는 셀입니다.

"연결 데이터를 최신 정보로 업데이트하기"를 선택하여 데이터를 추가하면 페이지가 자동으로 새로 고쳐지고 추가된 데이터가 표시됩니다.

![](<../../../../.gitbook/assets/image (1239).png>)

주문 세부 정보 변경 사항은 아래에 별도로 적용됩니다.

### 추가된 데이터만 전송하기&#x20;

서버 명령을 편집하고 파라미터 목록에서 주문 세부 정보 변경을 선택합니다. AddRows" 파라미터를 클릭하고 삽입을 두 번 클릭합니다. 루프의 하위 명령을 데이터 테이블 작업 명령으로 선택하고, 작업 유형을 추가하고, 대상 테이블을 주문 세부 정보 테이블로 선택하고, 업데이트 필드 및 값을 설정합니다.

![](<../../../../.gitbook/assets/image (1188).png>)

페이지에서 서버단 명령을 설정한 다음 페이지를 실행하고 페이지의 하 테이블에 두 줄의 새 데이터 행을 추가한 다음 주문 세부 정보 업데이트 버튼을 클릭하면 서버는 순환 명령의 데이터 테이블 작업 명령을 실행하고 변경된 데이터만 보내고 추가된 주문 테이블 세부 정보 데이터를 보내고 서버 측 데이터 테이블에 추가합니다.

![](<../../../../.gitbook/assets/image (163).png>)

### 수정된 데이터만 전송&#x20;

서버 명령을 편집하고 파라미터 목록에서 주문 세부 정보 변경을 선택합니다. EditRows" 파라미터 클릭하고 삽입을 두 번 클릭합니다. 루프의 하위 명령을 데이터 테이블 작업 명령, 작업 유형 업데이트, 대상 테이블 주문 세부 정보 테이블, 업데이트 필드 설정 및 업데이트 조건으로 선택합니다.

![](<../../../../.gitbook/assets/image (1183).png>)

페이지에서 호출 서버단 명령을 설정한 다음 페이지를 실행하고 페이지의 주문 세부 정보 테이블 테이블에서 데이터를 편집합니다.  다음 주문 세부 정보 업데이트 버튼을 클릭하면 서버단에서 반복 명령의 데이터 테이블 작업 명령을 실행하여 주문 테이블에서 수정된 주문 테이블 세부 정보 데이터를 보내고 서버단 데이터 테이블로 업데이트합니다.

### 삭제된 데이터만 전송

서버 명령을 편집하고 파라미터 목록에서 주문 세부 정보 변경을 선택합니다. DeleteRows® 파라미터를 클릭하고 삽입을 두 번 클릭합니다. 루프의 하위 명령을 데이터 테이블 작업 명령으로 선택하고, 작업 유형은 삭제하고, 대상 테이블은 주문 세부 정보 테이블이며, 업데이트 조건을 설정합니다.

![](<../../../../.gitbook/assets/image (1240).png>)
