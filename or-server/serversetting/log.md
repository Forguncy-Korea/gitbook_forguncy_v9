# 로그

포건시는 로그 보기를 지원합니다. 서버단에서 지정된 날짜부터 1일 이내에 지정된 애플리케이션의 로그를 볼 수 있습니다.



### 로그&#x20;

서버관리자에서 \[설정>로그]를 선택하여 로그관리페이지로 들어갑니다.

### 로그 설정&#x20;

로그 설정 탭에서 배포된 포건시 애플리케이션에 대해 로그를 구성할 수 있습니다.&#x20;

<figure><img src="../../.gitbook/assets/image (1852).png" alt=""><figcaption></figcaption></figure>



#### 로그 파일 구성&#x20;

보관할 최대 파일 수, 최대 보관 일수 및 로그 파일의 단일 파일 크기 제한을 설정합니다.

#### 로그 모듈 구성&#x20;

| 로그모듈           | 설정지침                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| -------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 예외 로그          | <p>애플리케이션에서 예외가 발생하면 예외 로그에 기록됩니다. 예외 로그를 ​​통해 애플리케이션 내부에서 발생한 오류를 확인하고 애플리케이션을 수정할 수 있습니다.</p><p>"활성화"를 선택한 후 모듈의 특정 레코드 내용을 선택할 수 있습니다.</p><p>포함: 요청 경로, 사용자 이름, 쿼리 매개변수, 요청 데이터, 예외 호출 스택, 요청 ID.</p><p>"필터"를 클릭하여 필터 로그 내용을 설정하고 포함 및 제외 키워드를 설정하십시오.</p><p><img src="https://help.grapecity.com.cn/download/attachments/80953070/image2022-9-22_11-29-37.png?version=1&#x26;modificationDate=1673923283000&#x26;api=v2&#x26;effects=border-simple,blur-border" alt=""><br></p> |
| HTTPRequestLog | <p>모든 Http 요청은 이 로그에 기록됩니다. Http 요청 로그는 웹 애플리케이션 디버깅, Http 트래픽 검사, 웹 애플리케이션에서 발생하는 상황 조사, 해커의 공격 감지에 유용합니다 .</p><p>"활성화"를 선택한 후 모듈의 특정 레코드 내용을 선택할 수 있습니다</p><p>포함: 경로, 매개변수, IP 주소, 방법, 호스트, Https인지 여부, 쿠키, 참조자, 요청 헤더, UserAgent, 요청 ID.</p><p>"필터"를 클릭하여 필터 로그 내용을 설정하고 포함 및 제외 키워드를 설정하십시오.</p>                                                                                                                                                                                |
| HTTPRespondLog | <p>완료되고 클라이언트에 응답된 모든 Http 요청은 이 로그에 기록됩니다. Http 응답 로그를 보면 사용자는 해당 요청의 결과와 소요 시간을 확인할 수 있습니다.</p><p>"활성화"를 선택한 후 모듈의 특정 레코드 내용을 선택할 수 있습니다</p><p>포함: 경로, 반환 코드, 시간, 사용자 이름, 콘텐츠 길이, 콘텐츠 유형, 응답 헤더, 요청 ID.</p><p>"필터"를 클릭하여 필터 로그 내용을 설정하고 포함 및 제외 키워드를 설정하십시오.</p>                                                                                                                                                                                                                  |
| 결과 데이터로그       | <p>일부 작업에는 이 로그에 기록되는 결과가 있습니다 .</p><p>"활성화"를 선택한 후 모듈의 특정 레코드 내용을 선택할 수 있습니다</p><p>포함: 경로, 성공, 정보, 요청 ID.</p><p>"필터"를 클릭하여 필터 로그 내용을 설정하고 포함 및 제외 키워드를 설정하십시오.</p>                                                                                                                                                                                                                                                                                                                 |
| 로그인 로그         | <p>로그인 성공 여부에 관계없이 모든 로그인 작업이 이 로그에 기록됩니다.</p><p>공통 인증 로그인, 싱글 사인온 , Windows 도메인 인증 로그인 및 타사 인증 로그인을 포함합니다.</p><p>"활성화"를 선택한 후 모듈의 특정 레코드 내용을 선택할 수 있습니다</p><p>포함: 경로, 성공, 실패 정보, 성공 정보, 요청 ID.</p><p>"필터"를 클릭하여 필터 로그 내용을 설정하고 포함 및 제외 키워드를 설정하십시오.</p>                                                                                                                                                                                                                            |
| 로그아웃 로그        | <p>모든 로그아웃이 이 로그에 기록 됩니다 .</p><p>"활성화"를 선택한 후 모듈의 특정 레코드 내용을 선택할 수 있습니다</p><p>포함: 경로, 성공, 실패 정보, 성공 정보, 요청 ID.</p><p>"필터"를 클릭하여 필터 로그 내용을 설정하고 포함 및 제외 키워드를 설정하십시오.</p>                                                                                                                                                                                                                                                                                                             |
| 비밀번호 분실 로그     | <p>잊어버린 모든 암호 작업은 이 로그에 기록됩니다.<br></p><p>"활성화"를 선택한 후 모듈의 특정 레코드 내용을 선택할 수 있습니다</p><p>포함: 경로, 성공, 실패 정보, 성공 정보, 요청 ID.</p><p>"필터"를 클릭하여 필터 로그 내용을 설정하고 포함 및 제외 키워드를 설정하십시오.</p>                                                                                                                                                                                                                                                                                                     |
| 메일 전송 오류       | <p>모든 전송 이메일은 이 로그에 기록됩니다.</p><p>"활성화"를 선택한 후 모듈의 특정 레코드 내용을 선택할 수 있습니다</p><p>포함: 성공, 제목, 보낸 사람, 받는 사람, 시간, 오류 메시지, 요청 ID.</p><p>"필터"를 클릭하여 필터 로그 내용을 설정하고 포함 및 제외 키워드를 설정하십시오.</p>                                                                                                                                                                                                                                                                                                 |
