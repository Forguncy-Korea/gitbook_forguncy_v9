# 웹사이트 배포

포건시빌더에 게시된 웹 사이트를 소개합니다.

디자이너의 리본 메뉴 모음에서 배포-> 서버 배포를 선택하여 배포 설정을 수행합니다. 설정 지침은 표에 나와 있습니다.

![](<../../.gitbook/assets/image (1759).png>)

| 설정                                   | 설명                                                                                                                                                                                                                                                                                                                                                                                                               |
| ------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 서버                                   | 포건시 측의 컴퓨터 이름, IP 주소 또는 서버 바인딩이 설치된 도메인 이름                                                                                                                                                                                                                                                                                                                                                                       |
| 사용자 이름                               | <p>Administrator를 사용할 수 있는 포건시 서비스 측의 사용자 이름을 지정합니다. Administrator에는 모든 권한이 있습니다.</p><p>사용자가 응용 프로그램을 게시하려면 해당 사용자에게 서버에 게시할 수 있는 추가 권한이 있어야 합니다. 사용 권한 설정은 역할 만들기를 참조하십시오.</p>                                                                                                                                                                                                                                  |
| 암호                                   | 사용자의 암호입니다. Administrator 사용자의 기본 암호는 123456.                                                                                                                                                                                                                                                                                                                                                                    |
| 사이트 포트                               | HTTP 기본값은 80이고 HTTPS 기본값은 443이며 수정할 수 있습니다.                                                                                                                                                                                                                                                                                                                                                                      |
| 응용 프로그램 이름                           | 웹 사이트의 이름입니다.                                                                                                                                                                                                                                                                                                                                                                                                    |
| 대상 URL                               | 웹 사이트의 URL입니다.                                                                                                                                                                                                                                                                                                                                                                                                   |
| 데이터베이스 덮어쓰기                          | <p></p><p>웹 사이트를 게시할 때 디자이너의 데이터베이스가 서버의 데이터베이스를 덮어쓸지 여부는 전체 및 반덮어링을 선택할 수 있습니다.</p><ul><li>모두 덮어쓰기 : 디자이너의 테이블 구조와 데이터로 서버 측을 완전히 덮어씁니다.</li><li>부분 덮어쓰기: 서버 테이블 구조를 디자이너의 테이블 구조로 덮어쓰고 서버 측 데이터를 유지합니다. 특정 데이터 병합 설정은 데이터 병합 및 마이그레이션을 참조하십시오.</li></ul><p>(포건시  V7 및 이전 버전의 경우 이 옵션은 포건시 내장 테이블에만 유효하며 SQL Server, Oracle 데이터베이스 등과 같은 아웃리치 데이터베이스에서는 작동하지 않습니다. 활자 V7.1은 아웃리치 데이터베이스의 반피치를 지원하기 시작했습니다.)</p> |
| 배포 후 서버를 온라인으로 전환                    | 이 옵션을 선택하면 웹 사이트가 게시될 때 사용할 수 있습니다.                                                                                                                                                                                                                                                                                                                                                                              |
| 사용자 서비스 배포                           | 빌더와 서비스 쪽의 사용자 관리는 분리되어 있으며 디자이너 사용자 관리의 역할, 사용자 및 조직 구조를 서버 측 사용자 관리에 게시하도록 선택할 수 있습니다.                                                                                                                                                                                                                                                                                                                         |
| 로컬의 페이지 접근 권한과 UI 사용 권한 설정을 서버에 덮어쓰기 | 이 옵션을 선택하면 디자이너에서 설정한 페이지 및 셀 권한이 서버에 게시되고 서버 쪽 페이지 및 셀 사용 권한이 재정의됩니다. 자세한 내용은 페이지 및 셀 권한을 참조하십시오.                                                                                                                                                                                                                                                                                                               |
| HTTPS                                | HTTPS 웹 사이트에 게시 된 웹 사이트, HTTPS 인증서가 있는지 확인하고 웹 사이트가 제대로 작동하는지 확인하기 위해이 옵션을 선택하십시오.                                                                                                                                                                                                                                                                                                                               |

## 연결 테스트&#x20;

설정이 완료되면 연결 테스트를 클릭합니다. 성공하면 다음과 같은 대화 상자가 나타나고 \[설정 저장]을 클릭하여 자주 사용하는 설정을 저장할 수 있습니다.

![](<../../.gitbook/assets/image (485).png>)

## 웹사이트 배포&#x20;

게시를 클릭하여 현재 포건시  응용 프로그램을 서버에 배포하고 릴리스 준비 중에 진행률 표시줄이 사용자에게 현재 진행률을 알려줍니다.

배포된 앱이 관리 콘솔에 이미 있는 경우 이전 앱을 덮어쓰기 위해 릴리스를 계속하라는 프롬프트 상자가 나타나고 마지막으로 릴리스된 시간과 앱 버전이 표시됩니다. 게시를 계속하거나 게시를 취소하도록 선택할 수 있습니다.

![](<../../.gitbook/assets/image (479).png>)
