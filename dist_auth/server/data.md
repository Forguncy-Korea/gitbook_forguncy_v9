# 데이터 권한

서버관리자에서 애플리케이션의 모든 데이터 테이블에 대한 권한을 설정할 수 있습니다.

서버관리자 진입 후 "허가"를 선택하고 이동형 애플리케이션 목록에서 애플리케이션을 선택한 후 "데이터 권한"을 클릭하면 현재 애플리케이션에서 권한이 활성화된 모든 데이터 테이블과 권한 설정을 볼 수 있습니다. .

데이터 권한을 설정한 후 "설정 저장"을 클릭하면 권한이 적용됩니다.



<figure><img src="../../.gitbook/assets/image (1102).png" alt=""><figcaption></figcaption></figure>

### 사용 권한 테이블 선택&#x20;

사용 권한 테이블 선택을 클릭하여 "테이블 열기 권한 선택"

<figure><img src="../../.gitbook/assets/image (86).png" alt=""><figcaption></figcaption></figure>

이 대화 상자에는 응용 프로그램의 모든 데이터 테이블이 나열됩니다. 데이터 테이블의 생성 레코드 권한, 행 권한 및 필드 권한을 설정할 수 있습니다. 확인 후 해당 권한을 설정할 수 있습니다.

<figure><img src="../../.gitbook/assets/image (625).png" alt=""><figcaption></figcaption></figure>

### 데이터 권한 설정&#x20;

데이터 권한에는 데이터 생성 권한,  행 권한 및 열 권한이 포함됩니다.

* 행 권한: 사용자가 작업할 수 있는 행을 제어합니다.
* 열 권한: 사용자가 작업할 수 있는 열(필드)을 제어합니다.
* 데이터 생성 권한: 사용자 또는 역할이 데이터를 생성할 수 있는 권한을 제어합니다.

역할 열을 클릭하고 팝업 역할 목록에서 데이터/행/열 생성 권한을 부여하는 역할을 선택합니다.

<figure><img src="../../.gitbook/assets/image (331).png" alt=""><figcaption></figcaption></figure>

조건 필터 열을 클릭하여 조건을 설정합니다.

<figure><img src="../../.gitbook/assets/image (1396).png" alt=""><figcaption></figcaption></figure>

모든 기록을 설정하거나 사용자 정의할 수 있습니다.

<figure><img src="../../.gitbook/assets/image (792).png" alt=""><figcaption></figcaption></figure>

#### 데이터 생성 권한 만들기 <a href="#id-shu-ju-quan-xian-chuang-jian-ji-lu-quan-xian" id="id-shu-ju-quan-xian-chuang-jian-ji-lu-quan-xian"></a>

사용자 또는 역할이 데이터를 생성할 수 있는 권한을 제어합니다. 작업이 추가되었으며 변경할 수 없습니다.

#### 작동 권한 <a href="#id-shu-ju-quan-xian-hang-quan-xian" id="id-shu-ju-quan-xian-hang-quan-xian"></a>

사용자가 조치를 취할 수 있는 행을 제어합니다. 행 권한 작업에는 보기, 편집 및 삭제가 포함됩니다.



**하위 테이블 권한 설정**

버튼을,클릭하여 하위 테이블 권한을 설정할 수 있습니다.

팝업 하위 테이블 권한 설정 대화 상자에 데이터 테이블의 모든 하위 테이블이 나열됩니다. 데이터 테이블 옆의 확인란을 선택하면 테이블이 기본 테이블의 권한 설정을 사용합니다.
