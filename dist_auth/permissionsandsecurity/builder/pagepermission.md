# 페이지 권한

각 페이지에 대한 사용자/그룹에 대한 액세스를 설정합니다. 앱 개발 단계에서 빌더에서 페이지 권한을 설정하고 앱을 게시한 후 관리 콘솔에서 직접 페이지 권한을 설정할 수 있습니다.



## 빌더에서 페이지 권한 설정하기&#x20;

리본의 메뉴 모음에서 보안-> 권한관리를 선택하고 권한 설에서 역할을 선택하고 해당 역할에 액세스할 수 있는 페이지를 선택합니다.

<figure><img src="../../../.gitbook/assets/image (326).png" alt=""><figcaption></figcaption></figure>



### 역할별 권한 설정&#x20;

"역할별 권한 설정" 모드를 선택하면 현재 애플리케이션의 모든 역할이 나열됩니다. 역할을 선택하면 현재 역할이 소유한 페이지 권한이 오른쪽 영역에 나열됩니다.

페이지 리소스 권한을 보기 위해 역할을 전환하고 페이지 앞의 체크박스를 선택하거나 선택 취소할 수 있습니다.

<figure><img src="../../../.gitbook/assets/image (609).png" alt=""><figcaption></figcaption></figure>

### 권한 설정 기준 페이지&#x20;

"권한"설정 기준페이지"    모드를 선택하면 현재 애플리케이션의 모든 페이지(내장 페이지 제외)가 나열됩니다. 페이지를 선택하면 현재 페이지에 액세스할 수 있는 역할이 오른쪽에 나열됩니다.

페이지를 전환하여 액세스가 허용된 역할을 볼 수 있으며 역할 앞의 확인란을 선택하거나 선택 취소할 수 있습니다.

<figure><img src="../../../.gitbook/assets/image (1610).png" alt=""><figcaption></figcaption></figure>

예를 들어 페이지 권한을 관리자 역할로 설정한 사용자는 아래 그림과 같이 주문 목록 페이지에 접근할 수 있습니다.



![](<../../../.gitbook/assets/image (493).png>)

역할을 설정하지 않은 사용자가 로그인 하였을 경우, 401 페이지가 표시됩니다.
